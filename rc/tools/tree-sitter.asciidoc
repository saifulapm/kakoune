= Tree-sitter

Native tree-sitter integration providing syntax-aware highlighting,
indentation, text objects, navigation, code folding, split/join,
symbol navigation, and AST data exposed to kakscript.

Query files for 245+ languages are bundled from Helix upstream.
Injection layers are detected recursively (matching Helix's algorithm),
enabling nested language injection (e.g. kak-into-kak self-injection,
HTML → JavaScript → regex).

== Grammar Management

Grammars are compiled shared libraries (`.so`/`.dylib`) built from
tree-sitter grammar source repositories. Grammars are auto-installed
on first use — when you open a file, if the grammar is not yet compiled,
it will be fetched and built automatically.

*tree-sitter-install* [langs...]::
    install grammars; without arguments installs for current buffer's
    language; with arguments installs specific languages (no matching
    buffer required)

*tree-sitter-update* [langs...]::
    re-fetch and rebuild grammars; without arguments updates current
    buffer's language; with arguments updates specific languages

*tree-sitter-remove* <langs...>::
    remove user-installed grammars

*tree-sitter-install-list*::
    show all installed grammars

*tree-sitter-install-all*::
    install all grammars defined in `tree-sitter-grammars.kak`

*tree-sitter-update-all*::
    update all currently installed grammars

Grammar definitions are in `rc/tools/tree-sitter-grammars.kak`. Each
language has a filetype hook that sets the source repository and revision:

-----
hook global BufSetOption filetype=rust %{
    set-option buffer tree_sitter_lang "rust"
    set-option buffer tree_sitter_source "https://github.com/tree-sitter/tree-sitter-rust"
    set-option buffer tree_sitter_rev "261b20226c04..."
    set-option buffer tree_sitter_subpath ""
}
-----

To override a grammar source, add your own hook in kakrc (runs after autoload).

== File Locations

Lookup order (config overrides runtime):

1. `%val{config}/runtime/grammars/` and `%val{config}/runtime/queries/<lang>/`
2. `%val{runtime}/runtime/grammars/` and `%val{runtime}/runtime/queries/<lang>/`

Grammars are installed to `%val{config}/runtime/grammars/`.
Bundled queries ship in `%val{runtime}/runtime/queries/`.

Query files follow Helix/Neovim naming conventions:

- `highlights.scm` — syntax highlighting (required)
- `indents.scm` — indentation rules
- `textobjects.scm` — text objects (function, class, parameter, etc.)
- `locals.scm` — local variable scoping
- `injections.scm` — embedded language injection
- `folds.scm` — foldable regions
- `tags.scm` — symbol definitions for navigation

== Enabling/Disabling

Tree-sitter is auto-enabled when a filetype is set. To manually control:

*tree-sitter-enable*::
    enable tree-sitter for current buffer

*tree-sitter-disable*::
    disable tree-sitter for current buffer

== Highlighting

Highlights are applied automatically from `highlights.scm`. Capture names
are converted to face names: `@keyword.function` becomes `ts_keyword_function`.
All faces are defined in `rc/tools/tree-sitter.kak` and can be customized.

Local variable scoping from `locals.scm` is also supported — local
definitions and references are highlighted with `ts_variable`.

=== Language Injection

Injection queries from `injections.scm` embed one language inside another
(e.g. JavaScript inside HTML, bash inside `%sh{...}` in kakscript).
Injections are detected recursively — sub-injections within injection
layers are discovered automatically (up to 8 levels deep), matching
Helix's algorithm. This enables kak-into-kak self-injection in block
bodies.

=== Rainbow Brackets

Nesting-depth-based bracket coloring using `tree-sitter-rainbow` highlighter.
Restricted to the visible range for performance.

-----
add-highlighter window/rainbow tree-sitter-rainbow
-----

Or auto-enable per window:

-----
hook global WinSetOption filetype=.+ %{
    try %{ add-highlighter window/rainbow tree-sitter-rainbow }
}
-----

Faces: `ts_rainbow_1` through `ts_rainbow_6` (cycled by nesting depth).

== Indentation

When `indents.scm` is available, auto-indent on newline is enabled
automatically. Manual reindentation is available via:

*tree-indent*::
    reindent the current selection based on the AST

*tree-indent-newline*::
    auto-indent the current line (used by the InsertChar hook)

Supported indent captures: `@indent`, `@indent.always`, `@outdent`,
`@outdent.always`, `@extend`, `@extend.prevent-once`, `@align`, `@anchor`.

The `#set! "scope" "all"` and `#set! "scope" "tail"` properties control
which lines an indent rule applies to.

== Text Objects

When `textobjects.scm` is available:

*tree-select* <object> <inside|around>::
    select the smallest text object at cursor

*tree-select-next* <object>::
    select the next text object after cursor

*tree-select-prev* <object>::
    select the previous text object before cursor

*tree-select-all* <object>::
    select all occurrences of a text object (multi-cursor)

*tree-filter* <object>::
    keep only selections that are inside the given text object

Common text objects: `function`, `class`, `parameter`, `comment`, `type`.

== AST Navigation

*tree-select-node*::
    select the AST node at cursor

*tree-parent*::
    select parent node

*tree-first-child*::
    select first child node

*tree-next-sibling*::
    select next sibling node

*tree-prev-sibling*::
    select previous sibling node

*tree-expand*::
    expand selection to parent node (with history stack)

*tree-shrink*::
    shrink selection back (exact inverse of expand)

== Code Folding

Folds use `replace-ranges` — the buffer is never modified. When folded,
the hidden lines are replaced with a summary: `+-- N lines: preview`.
Folding prefers `compound_statement` body blocks over full statement
nodes, keeping `else`/`catch` clauses visible.

*tree-fold*::
    fold the AST node at cursor (uses `folds.scm` when available)

*tree-unfold*::
    unfold at cursor position (works from the fold header line too)

*tree-fold-all* [object]::
    without argument: fold all `@fold` nodes from `folds.scm`;
    with argument: fold all occurrences of a text object

*tree-unfold-all*::
    remove all folds

The fold face `ts_fold` can be customized.

== Split/Join

Toggle between single-line and multi-line forms (like treesj.nvim):

*tree-split*::
    expand a single-line node to multi-line

*tree-join*::
    collapse a multi-line node to single line

*tree-toggle*::
    split if single-line, join if multi-line

== Symbol Navigation

When `tags.scm` is available:

*tree-symbols* [name]::
    without argument: show all symbol definitions in info popup;
    with argument: fuzzy-match and jump to a symbol (with completion)

*tree-goto-def* [name]::
    jump to the definition of the word under cursor (or given name)

== Sticky Context

*tree-update-context*::
    set `%opt{tree_context}` to the enclosing function/class name
    at the top of the visible window (hooked to `NormalIdle`)

To display in the modeline:

-----
set-option global modelinefmt '%opt{tree_context} %val{bufname} ...'
-----

== Cursor Query (kakscript integration)

*tree-query-cursor*::
    set options with tree-sitter information at cursor position,
    enabling kakscript commands to use tree-sitter data

Options set by `tree-query-cursor`:

- `tree_cursor_lang` — injection-aware language at cursor (e.g. `javascript` inside `<script>` in HTML)
- `tree_cursor_node_type` — leaf AST node type (e.g. `string_literal`, `identifier`)
- `tree_cursor_parent_type` — parent node type (e.g. `function_definition`)
- `tree_cursor_in_string` — `true` if cursor is inside a `@string*` highlight capture
- `tree_cursor_in_comment` — `true` if cursor is inside a `@comment*` highlight capture

Example: injection-aware comment toggle in kakscript:

-----
define-command comment-toggle %{
    tree-query-cursor
    evaluate-commands %sh{
        case "$kak_opt_tree_cursor_lang" in
            javascript|typescript|rust|cpp|c|go|java) echo "comment-line //" ;;
            python|ruby|bash) echo "comment-line #" ;;
            html|xml) echo "comment-block <!-- -->" ;;
            css) echo "comment-block /* */" ;;
            *) echo "comment-line #" ;;
        esac
    }
}
-----

== Inspection / Debugging

*tree-sitter-scopes*::
    show the ancestor node chain at cursor (includes injection layers)

*tree-sitter-highlight-name*::
    show the highlight capture name at cursor

*tree-sitter-subtree*::
    show the AST subtree at cursor as S-expression

*tree-sitter-layers*::
    show root language and injection layers at cursor

== User Mode

A `tree` user mode is defined with key mappings for common operations.
To use it, map it to a key:

-----
map global user t ': enter-user-mode tree<ret>'
-----

Key bindings:

[cols="10,90"]
|===
| Key | Command

| f/F | select function around/inside
| c/C | select class around/inside
| a/A | select parameter around/inside
| o   | select comment
| n/p | next/prev function
| s   | select node
| P   | parent
| i   | first child
| ]/[ | next/prev sibling
| e/E | expand/shrink
| z/Z | fold/unfold
| <a-z>/<a-Z> | fold all functions / unfold all
| j   | split/join toggle
| J   | split to multi-line
| <a-j> | join to single-line
| *   | select all functions
| /   | filter to functions
| ?   | show scopes
| h   | show highlight name
| t   | show AST subtree
| l   | show injection layers
|===

== Supported Predicates

Query files can use the following predicates:

- `#eq?`, `#not-eq?` — string/capture equality
- `#match?`, `#not-match?` — regex matching
- `#any-of?`, `#not-any-of?` — set membership
- `#not-kind-eq?` — node type filtering
- `#same-line?`, `#not-same-line?` — line position comparison
- `#one-line?`, `#not-one-line?` — single-line span check
- `#is?`, `#is-not?` — property predicates
- `#set!` — property setting (used for indent scope, injection language)

Helix/Neovim-specific predicates (`#strip!`, `#select-adjacent!`,
`#lua-match?`, `#gsub!`, `#any-not-eq?`) are silently ignored.
